# Project wiring

Each project should add a `docs/INDEX.md` that references this charter.

- Global protocol lives here.
- Project-specific overrides (channels, URLs, secrets locations) live in the project repo.

## Access registries (recommended)
For each app repo, keep a non-secret `docs/ACCESS.md` that states what systems (Vercel/Supabase/etc) are used and what tooling is pre-provisioned on the bot host. Link to bot-charter for process rules.
