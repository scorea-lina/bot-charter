# Status Updates Protocol (Org-wide)

## Goal
Minimize back-and-forth by making every update actionable.

## Required fields (copy/paste)
**Status:** <one line>
**Artifact:** <PR link | deploy link | screenshot | log link>
**ETA:** <clock time + timezone>
**Blocker (if any):** <smallest unblock needed>
**Next step:** <what you will do next>

## Rules
- **Concept of time:** avoid vague ranges like "1–4 hours". Use **Small/Medium/Large** and include the next check-in time ("I'll post an update at <time>").
  - Small: <30 min
  - Medium: 30–120 min
  - Large: >2h (must include interim checkpoint)
- If you reference an artifact, include the link.
- If you cannot provide an ETA, provide the *next checkpoint time* when you will update.
- Avoid vague statuses like "working on it" without an artifact or next checkpoint.
